leetcode 11 盛最多水的容器

2021-01-26
1分钟阅读时长
难题集
编程练习

题目描述：

给你 n个非负整数 a1，a2，…，an，每个数代表坐标中的一个点 (i, ai) 。在坐标内画 n 条垂直线，垂直线 i的两个端点分别为 (i, ai) 和 (i, 0) 。找出其中的两条线，使得它们与 x 轴共同构成的容器可以容纳最多的水。

说明：你不能倾斜容器。

示例：

输入：[1,8,6,2,5,4,8,3,7]
输出：49 
解释：图中垂直线代表输入数组 [1,8,6,2,5,4,8,3,7]。在此情况下，容器能够容纳水（表示为蓝色部分）的最大值为 49。

输入：height = [1,1]
输出：1

输入：height = [4,3,2,1,4]
输出：16

输入：height = [1,2,1]
输出：2

提示：

n = height.length
2 <= n <= 3 * 10e4
0 <= height[i] <= 3 * 10e4

这道题目为许久之前所做，但时至今日依然有些许的困难，索性就将其记录下来。

解题想法：

首先题目需要求解的是数组之间围成的矩形面积的最大值，而很明显这个矩形面积的最大值取决于长和宽的大小，放在这个题目中，首先限制的便是两边的数的大小，但是最终的长度依旧只取决于较小的那个数。其次就是这两个数在数组中的索引距离。因此我们需要在一个时刻拥有两边的数据（即左边界的数据以及右边界的数据），因此可以使用两个变量来分别指向这两个数据，也就引出了这道题所使用的方法，双指针法。

然后就要开始移动指针了，当然前提是，左指针的索引小于右指针的索引。在每一次的判断中，我们要移动的始终是两个指针中较小的那一个指针。这时候就会有疑问了，现在开始讲解。假设我们有左右两个指针l、r（l <r这时候指针指向的为值)此时我们移动较大的指针，也就是r，我们只会将指针往中间移动，因为在初始化指针的时候应该是在边界处。如果--r指向的值较小，那么由于矩形的长宽都变小了，那么肯定不会是最优解。如果--r指向的值较大，也会因为受到较小的l指针的限制而无法使面积增大，因此只有移动较小的指针才有获得更优解的可能。只要每次的循环都能更新最大值，遍历一次便是题目所需要的最优解。下面附上C++代码

class Solution {
public:
    int maxArea(vector<int>& height) {
        int left = 0, right = height.size() - 1;
        int field = 0;
        while (left < right) {
            field = max(field, min(height[right], height[left]) * (right - left));
            if (height[left] < height[right]) {
                ++left;
            }
            else {
                --right;
            }
        }
        return field;
    }
};
