leetcode 1128 等价多米诺骨牌对的数量

2021-01-27
1分钟阅读时长
错题集
编程练习

题目描述：

给你一个由一些多米诺骨牌组成的列表 dominoes。

如果其中某一张多米诺骨牌可以通过旋转 0 度或 180 度得到另一张多米诺骨牌，我们就认为这两张牌是等价的。

形式上，dominoes[i] = [a, b] 和 dominoes[j] = [c, d] 等价的前提是 a==c 且 b==d，或是 a==d 且 b==c。

在 0 <= i < j < dominoes.length 的前提下，找出满足 dominoes[i] 和 dominoes[j] 等价的骨牌对 (i, j) 的数量。

示例：

输入：dominoes = [[1,2],[2,1],[3,4],[5,6]]
输出：1

提示：

1 <= dominoes.length <= 40000
1 <= dominoes[i][j] <= 9

解题想法：

这道题在leetcode上是属于简单题。。然而却不能做出来。。

首先这道题乍一看要处理的东西很多，但是只要经过推敲不难看出，题目只需要判断二维数组中重复的单子数组对数。（因为多米诺骨牌只有两个数且只有反转的操作，因此可以将题目简化为一个判重问题，也就是这个判重问题卡住了我）。由于骨牌的数字不会超过9，也就是皆为个位数，由此要处理这个判重问题我们会希望如果能够忽略每一个单子数组内的元素顺序，就能够进行比较简单的判重，所以也就有了一个较为简单的方法，就是每次都将单子数组里的元素的较小值作为十位数，另一元素作为个位数，拼成一个数字。对一个数字进行判重就较为简单了。（由于这个拼成的数字最大值为99，所以只需要开一个100大小的数组便足够了），下面附上C++代码

class Solution {
public:
    int numEquivDominoPairs(vector<vector<int>>& dominoes) {
        vector<int> res(100); //用于保存对数的数组
        int sum = 0;
        for (auto item: dominoes) {
            int num = item[0] < item[1] ? item[0] * 10 + item[1] : item[1] * 10 + item[0];
            sum += res[num];
            ++res[num];
        }
        return sum;
    }
};
