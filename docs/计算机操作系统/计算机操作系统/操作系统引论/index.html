操作系统

一、操作系统的目标和作用

1.1 操作系统的目标

1.1.1 方便性

系统可以使用编译命令将用户采用高级语言书写的程序翻译成机器代码，或者直接通过OS所提供的各种命令操纵计算机系统。

1.1.2 有效性

第一层含义是提高系统资源的利用率。另一层含义是提高系统的吞吐量。

1.1.3 可扩充性

无结构→模块化结构→层次化结构→微内核结构，微内核结构能方便地增添新的功能和模块，以及对原有的功能和模块进行修改，具有可扩充性。

1.1.4 开放性

所谓开放性，是指系统能够遵循世界标准规范，特别试遵循开放系统互连OSI国标标准。

1.2 操作系统的作用

1.2.1 OS作为用户与计算机硬件系统之间的接口

如图所示，用户通过三种方式使用计算机，即命令方式、系统调用方式和图标-窗口方式。

1.2.2 OS作为计算机系统资源的管理者

资源可以分为四类：处理机、存储器、IO设备和文件（数据和程序）

处理机管理是用于分配和控制处理机；存储器管理主要负责内存的分配与回收；IO设备管理是负责IO设备的分配（回收）与操纵；文件管理是用于实现队文件的存取、共享和保护

1.2.3 OS实现了对计算机资源的抽象

对于一台完全无软件的计算机系统（即裸机），它向用户提供的仅是硬件接口（物理接口）。

对计算机资源的抽象可以由以下构成：物理接口→IO软件→管理文件的软件→面向用户的窗口软件

二、操作系统的发展过程

2.1 未配置操作系统的计算机系统

2.1.1 人工操作方式

事先穿孔的纸带作为程序和数据输入计算机。缺点：1、用户独占全机。2、CPU等待人工操作

2.1.2 脱机输入输出方式

由外围机将用户程序和数据输入到磁带上，当CPU需要这些程序和数据时，再从磁带上高速的调入内存。反之，则由CPU将数据从内存高速的输入磁带上，再通过外围机将结果通过相应的输出设备输出。

优点：减少了CPU的空闲时间、提高了IO速度

2.2 单道批处理系统

2.2.1 处理过程

为了实现对作业的连续处理，需要先把一批作业以脱机方式输入到磁带上，并在系统中配上监督程序，在它的控制下，时这批作业能一个接一个的连续处理。
