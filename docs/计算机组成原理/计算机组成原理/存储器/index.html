存储器

2021-03-13

计算机组成原理

一、存储器分类

1.1 按存储介质分类

存储介质是指能寄存“0”“1”两种代码并能区别两种状态的物质或元器件。通常指半导体、磁性材料和光盘

1.1.1 半导体存储器

现代半导体存储器一般采用超大规模集成电路，具有体积小、功耗低、存取时间短的优点。**缺点是当电源消失时，所存信息也随机丢失，是一种易失性存储器。**照材料不同也可以分为双极型和MOS半导体存储器，前者具有高速的特点，后者具有高集成度的特点，制造简单成本低廉功耗小

1.1.2 磁表面存储器

利用磁头进行读/写操作。可分为磁盘、磁带和磁鼓。按照剩磁状态的不同来区分“0”或“1”，剩磁状态不会轻易丢失，具有非易失型特点

1.1.3 磁芯存储器

环状，也属于不易失的永久记忆存储器

1.1.4 光盘存储器

也是非易失存储器，应用越来越广

1.2 按存储方式分类

可分为随机、只读、顺序和直接存取

1.2.1 随机存储器（RAM）

特点是任何一个存储单元的内容都可以随机存取，且存取时间与存储单元的物理位置无关。计算机主存都采用RAM，RAM又分为静态RAM （触发器原理） 以及动态RAM （电容充放电原理）

1.2.2 只读存储器（ROM）

对存储内容只能读出而不能重新写入的存储器。通常存放固定不变的程序、常数和字库或操作系统的固化。与RAM共同作为主存的一部分，统一构成主存的地址域后续又诞生了 可编程只读存储器（PROM）、可擦除可编程只读存储器（EPROM）、用电可擦除可编程只读存储器（EEPROM）以及闪存存储器具有EEPROM的特点，但是速度快很多

1.2.3 串行访问存储器

需要按照物理位置的先后顺序寻找地址，如磁带存储器。这类存储器又被称为顺序存取存储器。而如磁盘需要首先直接指出某个区域，在进行顺序访问，则称为直接存取存储器

1.3 按作用分类

主要分为主存、辅存、缓存。

主存主要特点是可以与CPU直接交换信息。辅存是主存的后援，存放暂时不用的程序数据，不能与CPU直接交换信息。缓存用于两个速度不同的存储器中。存储器分类如图

Note：

1、缓存-主存层次主要解决CPU和主存速度不匹配的问题。主存和缓存之间的数据调动是由硬件自动完成的，对程序员是透明的。

2、主存-辅存层次主要解决存储系统的容量问题。 当CPU需要辅存的信息时，将辅存内容调入主存。 主存和辅存之间的数据调动是由硬件和操作系统共同完成的

3、缓存-主存这一层次的速度接近于缓存，高于主存。主存-辅存这一层次的不断发展，形成了虚拟存储系统。在这个系统中程序员编程的地址范围与虚拟存储器的地址空间相对应。

三、主存储器

3.1 工作原理

当要从主存储读出某一信息字时，首先由CPU将该字的地址送到MAR，经地址总线送至主存，然后发出读命令。主存街道读命令后，得知需将该地址单元的内容读出，便进行读操作。将该单元的内容读至数据总线上。若要向主存存入一个信息字时，首先CPU将该字所在的主存单元的地址经MAR送到地址总线，并将信息字送入MDR，然后向主存发出写命令，主存接到写命令后，便将数据线上的信息写入到对应地址线指出的主存单元中。

3.2 主存的技术指标

3.2.1 存储容量

存储容量=存储单元个数 * 存储字长

存储容量=存储单元个数 * 存储字长 / 8

3.2.2 存储速度

由存储时间和存取周期来表示

存取时间（访问时间），启动一次存储器操作到完成该操作所需的全部时间

存取周期，存储器进行连续两次独立的存储器操作所需的最小间隔时间。

3.2.3 存储器带宽

单位时间内存储器存取的信息量

3.3 改善传输速度

1、缩短存储周期

2、增加存储字长，使每个存取周期能够读写更多的二进制位数

3、增加存储体

四、半导体存储芯片

4.1 半导体存储芯片的基本结构

地址线和数据线的位数共同反映存储芯片的容量，如地址线为10根，数据线为4根。芯片容量为$2^{10} * 4 = 4K$位，控制线主要有读写控制线和片选线。（读写控制线数为1或2，片选线通常为1，也可能为2），因此芯片的最少线数应由这三者组成

4.2 译码驱动方式

主要方法有两种，线选法和重合法，其中线选法一次选一个字，重合法一次选一个位。

4.3 动态RAM的刷新

刷新是一行一行进行的，必须在刷新周期内，才能保证动态RAM的信息不丢失，通常有三种方式，集中刷新，分散刷新，异步刷新

4.3.1 集中刷新

是在规定的一个刷新周期内，对全部存储单元集中一段时间逐行进行刷新，此时必须停止读写操作。但是存在“死时间”又称为访存“死区”。如图

4.3.2 分散刷新

对每行存储单元的刷新分散到每个存取周期内完成。其中把存取周期分为两段，前半段用来读写或维持信息，后半段用来刷新。不存在死时间，但是存取周期变长了，系统速度降低如图

4.3.3 异步刷新

在一个固定的刷新周期内对每一行都进行一次刷新，即每隔同一时间段对每行刷新。如图

4.4 动态RAM与静态RAM的比较

1、动态RAM的集成度要远高于静态RAM

2、动态RAM行列地址按先后顺序输送，奸商了芯片引脚，封装尺寸也减少

3、动态RAM功耗比静态RAM小

4、动态RAM价格比静态RAM便宜

五、只读存储器

5.1 掩模ROM

出厂状态即为最终状态，用户是无法改变的

5.2 PROM

是可以实现一次性编程的只读存储器。熔丝断掉出将读出0，熔丝未断处将读出1

5.3 EPROM

是一种可擦除可编程只读存储器，可以由用户对其所存信息作任意次的改写

六、提高访存速度的措施

6.1 单体多字系统

可以在一个存取周期内，从同一地址取出多条指令，然后再逐条指令送到CPU执行。如图

6.2 多体并行系统

并行工作即同时访问N各模块，同时启动，同时读出。

6.2.1 高位交叉编址

程序按体内地址顺序存放，一个体存满后，再存入下一个体。高位地址可表示体号，地位地址为体内地址。 如图

6.2.2 低位交叉编址

程序连续存放在相邻体中，有交叉存储之称，低位地址用来表示体号，高位地址为体内地址。在模4交叉编址中，最低两位地址即可判断存储于哪个体 如图

Note：

假设每个体的存储字长和数据总线一致，并假设低位交叉的存储器模块数位n存取周期为T，总线传输周期为t，那么采用流水线方式存储时，应该满足$T=nt$,为了保证启动某体后，经$nt$时间再次启动该体时，它的上次存取操作已完成，要求低位交叉存储器的模块数大于或等于n。流水线方式如图。由此可见对于低位交叉的存储器，连续读取n个字所需的时间为： $T+(n-1)t$，若采用高位交叉编址，则连续读取n个字的时间为$nt$

七、高速缓冲处理器

7.1 Cache的一些常规参数

1、在一个程序执行期间，设$N_c$为访问Cache的总命中次数，$N_m$为访问主存的总次数，则命中率h为$h={N_c\over{N_c + N_m}}$

2、设$t_c$为命中时的Cache访问时间，$t_m$为未命中时主存访问时间，1-h表示未命中率，则Cache-主存系统的平均访问时间$t_a$为$t_a={ht_c+(1-h)t_m}$

3、Cache-主存系统的平均访问时间$t_a$越接近于$t_c$越好，用e表示访问效率则有$e={t_c\over t_a}*0.01={t_c\over{ht_c+(1-h)t_m}}*0.01$

7.2 Cache的基本结构

Note：

当CPU想要读取某一存储块时，将经历以下过程，首先对于一个块的块内地址，可以直接对应到Cache中的块内地址，而块号通过地址映射变换机构对应到Cache的块号中，如果这个块命中了Cache，那么就直接访问Cache。如果未命中，那么将要判断这个块是否可以装进Cache中（Cache是否为满），如果无法装进，那么就要通过替换机构启动替换算法将这个块换进Cache中。当然，CPU只能这一次只能从主存中获得这个块了。

如果CPU要写入一个块，那么就通过两种写法分别有不同的效果，将于下文讲解。

7.3 Cache的两种写操作

7.3.1 写直达法

在写操作时，数据既写入Cache又写入主存。它能随时保证主存和Cache的数据始终一致，但增加了访存次数

7.3.2 写回法

即写操作时，只把数据写入Cache而不写入主存，但当Cache数据被替换出去时才写回主存。这个方法不能保证主存和Cache中的数据一致，而且为了识别Cache中的数据是否与主存一致，Cache中的每一块要增设一个标志位。

7.3.3 对于Cache写不命中的处理方法

7.3.3.1 写分配法

加载主存中的块到Cache中，然后更新这个Cache块。缺点是每次不命中都需要从主存中读取一块。

7.3.3.2 非写分配法

只写入主存不进行调块

7.4 Cache的改进

7.4.1 单一缓存和两级缓存

单一缓存–在CPU和主存之间只设一个缓存（与CPU制作在同一个芯片内，片内缓存），能够提高外部总线的利用率。

片外缓存–比主存动态RAM和ROM的存取速度更快，而且不使用系统总线作为传输路径，使用一个独立的数据路径。

7.4.2 统一缓存和分立缓存

统一缓存–指令和数据都存放在统一缓存内的Cache

分立缓存–指令和数据分别存放在两个缓存中，一个称为指令Cache，一个称为数据Cache。
