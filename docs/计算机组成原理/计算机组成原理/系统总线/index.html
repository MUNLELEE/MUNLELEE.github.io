计算机组成原理

一、总线的基本概念

计算机系统的五大部件之间的互连方式有两种，一种是个部件之间用单独的连线，称为分散连接，另一种是将个部件连到一组公共信息传输线上，称为总线连接。早期IO与存储器交换信息时，都需经过运算器，致使运算器停止运算，严重影响了CPU的工作效率。总线是连接多个部件的信息传输线，是各部件共享的传输介质。 （当多个部件与总线相连时，如果出现两个或两个以上部件同时向总线发送信息，势必导致信号冲突。因此在某一时刻，只允许有一个部件向总线发送信息，而多个部件可以同时从总线上接收相同的信息）

单总线结构与双总线结构相比，单总线当IO设备与主存交换信息时，原则上不影响CPU的工作，CPU仍可以继续处理不访问主存或IO设备的操作，使CPU效率有所提高。但是，因为只有一组总线，当某一时刻各部件都要占用总线时，就会发生冲突。因此需设计总线判优逻辑，而三总线开辟了一条CPU与主存之间的总线，称为存储总线。既提高了传输效率，又减轻了系统总线的负担。

二、总线的分类

2.1 片内总线

芯片内部的总线，远快于片外总线

2.2 系统总线

可以分为三类，数据总线、地址总线和控制总线

2.1.1 数据总线

数据总线用来传输各功能部件之间的数据信息，是双向传输总线，位数与机器字长、存储字长有关 （如果数据总线的宽度位8位，指令字长为16位，那么CPU在取值阶段必须两次访问主存）

2.1.2 地址总线

地址总线主要用来指出数据总线上的源数据或目的数据在主存单元的地址或IO设备的地址。地址线的位数与存储单元的个数有关。
